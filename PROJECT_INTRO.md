# PRD Sherpa - AI 驱动的产品需求管理助手

## 💡 解决的痛点

### 产品经理的困境
- **需求散落各处**：设计稿在 Figma、讨论在钉钉、文档在语雀，信息碎片化严重
- **PRD 写作耗时**：整理需求、编写文档至少需要 2-3 天，重复劳动多
- **需求理解偏差**：开发看不懂 PRD，产品解释不清楚，来回沟通成本高
- **历史需求难查**：想找半年前的某个功能设计，翻遍文档找不到
- **需求变更混乱**：多次迭代后，哪些功能已实现、哪些还在规划，一团乱麻

### 开发人员的痛苦
- **理解需求困难**：PRD 写得模糊，不知道产品到底想要什么
- **逻辑核对麻烦**：不确定自己的实现是否符合产品意图，反复确认
- **功能回溯困难**：想看某个功能最初的设计思路，找不到历史记录
- **边界条件不明**：异常情况怎么处理？PRD 里没写清楚

---

## ✨ PRD Sherpa 如何解决

### 1️⃣ **一站式需求汇总**
📤 **上传任何形式的资料**
- 设计稿（Figma 导出、PNG/JPG）
- 历史 PRD（PDF、Word、Markdown）
- 原型图、演示 PPT、竞品分析
- 会议纪要、讨论截图

🤖 **AI 自动提取关键信息**
- 功能点、用户故事、技术约束
- UI 规范、交互流程、数据模型
- 业务规则、异常处理、性能要求

📚 **生成结构化知识库**
- 12 个专业章节（项目概览、功能架构、用户体验、技术架构...）
- 自动分类整理，随时搜索查询
- 历史需求永久保存，永不丢失

---

### 2️⃣ **对话式需求澄清**
💬 **和 AI 聊天，快速写 PRD**
- 产品：「我想加一个用户登录功能」
- AI：「好的，请问登录方式是手机号+验证码，还是账号密码？是否需要第三方登录（微信/支付宝）？」
- 产品：「手机号验证码，要支持微信登录」
- AI：「明白了。验证码有效期多久？登录失败几次后需要限制？微信登录失败时如何处理？」

✅ **AI 会主动追问边界条件**
- 异常情况怎么办？
- 权限如何控制？
- 数据如何校验？
- 性能指标是什么？

📝 **对话完成，PRD 自动生成**
- 结构化的 Markdown/Word/PDF 文档
- 包含功能描述、流程图、数据模型
- 开发可以直接按照 PRD 实现

---

### 3️⃣ **开发核对逻辑的利器**
🔍 **智能问答，秒速查询**

**场景 1: 核对实现逻辑**
```
开发: "用户连续登录失败 5 次后，应该锁定账号多久？"
AI: "根据 PRD 第 3.2 节，连续失败 5 次后锁定 30 分钟。
     锁定期间显示：'账号已锁定，请 30 分钟后重试'。"
```

**场景 2: 查找历史功能**
```
开发: "上个版本的积分规则是怎么设计的？"
AI: "v2.3 版本的积分规则如下：
     - 每日签到 +10 积分
     - 分享商品 +5 积分
     - 完成订单 +订单金额的 1%
     详细规则见知识库-业务规则章节。"
```

**场景 3: 理解产品意图**
```
开发: "购物车为空时，'结算'按钮应该置灰还是隐藏？"
AI: "根据 PRD 原型图，应该置灰显示，点击时提示'请先添加商品'。
     这样做的目的是引导用户操作，而不是让用户困惑。"
```

**场景 4: 补充未明确的细节**
```
开发: "用户头像上传失败时应该如何处理？"
AI: "PRD 中未明确说明。根据类似场景（附件上传失败）的处理方式，
     建议：保留默认头像，显示错误提示，允许重试。
     是否需要和产品确认？"
```

---

### 4️⃣ **需求历史追溯**
📂 **所有对话自动归档**
- 每次讨论都保存到知识库
- 按时间线查看需求演变
- 了解功能的来龙去脉

🔄 **需求版本对比**（规划中）
- 查看 v1.0 → v2.0 改了什么
- 哪些功能被砍掉了，为什么？
- 避免重复造轮子

## 🚀 使用说明（5 分钟上手）

### 第一步：上传项目资料
1. 创建新项目：「电商 App v3.0」
2. 上传资料：
   - 设计稿（Figma 导出的 PNG）
   - 历史 PRD（v2.0 的 Word 文档）
   - 竞品分析（PDF）
   - 原型图（PPT）
3. 点击「构建知识库」，AI 自动分析整理（约 2-3 分钟）

### 第二步：对话澄清需求
1. 进入「对话」页面
2. 描述新功能：
   ```
   "我想加一个会员等级系统，根据用户消费金额自动升级。
   普通会员、银卡、金卡、钻石卡，享受不同折扣。"
   ```
3. AI 会追问细节：
   - 升级条件是什么？
   - 各等级享受什么权益？
   - 等级是否会降级？
   - 消费金额如何计算？
4. 回答完成后，点击「生成 PRD」

### 第三步：导出和分享
1. 选择格式：Markdown / Word / PDF
2. 下载 PRD 文档
3. 分享给开发团队

### 第四步：开发使用（高级用法）
**方式 1：网页查询**
- 打开项目知识库
- 搜索关键词：「登录」「积分」「支付」
- 查看相关设计和逻辑

**方式 2：对话询问**
- 新建对话
- 直接问：「购物车删除商品后如何刷新总价？」
- AI 基于 PRD 给出准确答案

**方式 3：核对逻辑**
- 开发写完代码后
- 问：「我的登录流程实现是否符合 PRD？」
- AI 对比 PRD，指出差异

- ⚡ **需求理解速度**：从 1 小时到 10 分钟

### 团队协作

- 🔄 **新人上手**：查历史知识库，快速了解业务

## 💼 典型使用场景

### 场景 1：新功能开发
1. 产品上传竞品分析、原型图
2. 和 AI 对话澄清需求
3. 生成 PRD 给开发
4. 开发参考知识库实现

### 场景 2：功能迭代
1. 查找历史 PRD（v1.0）
2. 对话说明改动：「登录改成手机号+密码」
3. AI 对比差异，生成增量 PRD
4. 开发只改必要的代码

### 场景 3：Bug 修复
1. 开发发现登录逻辑不明确
2. 在知识库搜索「登录」
3. 查看原始设计意图
4. 或直接问 AI：「连续失败 3 次后应该怎么处理？」

### 场景 4：新人培训
1. 新来的开发不了解业务
2. 打开项目知识库
3. 阅读「项目概览」「功能架构」章节
4. 有问题直接问 AI：「积分系统是怎么设计的？」

### 场景 5：需求评审
1. 产品分享 PRD 链接
2. 开发/测试在线查看
3. 不明白的地方直接在对话里问
4. AI 基于 PRD 回答，产品确认即可

## 🛠️ 核心技术

- **多模态 AI**：支持文本+图片分析（Gemini/GPT-4/Claude）
- **智能问答**：基于知识库的上下文理解
- **流式对话**：实时响应，ChatGPT 风格体验
- **知识图谱**：12 个维度结构化存储
- **语义搜索**：不只是关键词，理解语义
